Command: GREP - Global Regular Expression Print

The grep command is using to search string inside the file/files.

Options Description Example
–color successful matches will be highlighted grep –color book dummy.txt
-i, –ignore-case ignore case sense grep –color -i book dummy.txt
-v, –invert-match select non-matching lines grep -v book dummy.txt
-w, –word-regexp select those line who matches the whole word grep -w book dummy.txt

-h without filename grep -h book dummy.txt

The find command is using to search files and directories

Use Commands

Find Files Without 777 Permissions find / -type f ! -perm 777

Find and remove Multiple File find . -type f -name “*.txt” -exec rm -f {} \;
Find all Empty Files find /tmp -type f -empty
Find all Empty Directories find /tmp -type d -empty
File all Hidden Files find /tmp -type f -name “.*”
Find all Files Based on Group find /home -group developer

Find 50MB Files find / -size 50M
Find Size between 50MB – 100MB find / -size +50M -size -100M

Command: AWK

AWK is a line-oriented language basically used to manipulating data and generate reports. awk script no required any compiler to compile. In awk we use variables, numeric functions, string function and logical operators.

Awk is abbreviated from the names of the developers – Aho, Weinberger, and Kernighan.

An awk program operates on each line of an input file. Basic components are

  • BEGIN { …. initialization awk commands …}
  • { …. awk commands for each line of the file…}
  • END { …. finalization awk commands …}
ls -l | awk 'BEGIN {sum=0} {sum=sum+$5} END {print sum}'

Built In Variables In Awk

Variables Description
$1,$2… $n fields variables [$0 for entire line]
NR Number Row - keeps the current count of the row
NF Number field - keeps the current count if the fields
FS field separator [The default is “white space” ]
RS Record separator [The default is “New line character”]
OFS Specifies the Output separator.
ORS Specifies the Output separator.
ARGC Retrieves the number of passed parameters.
ARGV Retrieves the command line parameters. ARGV[1],ARGV[2]….
ENVIRON Array of the shell environment variables and corresponding values.
IGNORECASE To ignore the character case.

AWK control statements include:

  • if (condition) statement [ else statement ]
  • while (condition) statement
  • do statement while (condition)
  • for (expr1; expr2; expr3) statement
  • for (var in array) statement
  • break
  • continue

Command: Crontab

Crontab execute the command at a specific time regularly

Options Description
-e modify the crontab of current user
-l view current crontab
-r delete you crontab

Crontab Syntax

<Minutes> <Hours> <Day of month> <Month of year> <Day of week (0-6,Modify: 0 = Sunday)> <COMMAND>

Crontab Example

  • * * * * * –> every minute
  • 47 15 * * * touch /home/bob/movies.txt –> create the file movies.txt every day at 15:47
  • 30 5 1-15 * * –> at 5:30 in the morning, from the 1st to 15th each month
  • 0 0 * * 1,3,4 –> at midnight on Mondays, Wednesdays and Thursdays
  • 0 */2 * * * –> every two hours
  • */10 * * * 1-5 –> every 10 minutes Monday to Friday
  • @yearly –> equal to 0 0 1 1 *
  • @daily –> equal to 0 0 * * *
  • @hourly –> equal to 0 * * * *
  • @reboot Run at startup.
