AWS Databases

Mon, Jul 24, 2017

AWS offers a wide range of database services to fit your application requirements. These database services are fully managed and can be launched in minutes with just a few clicks.

Databases

  • RDS - OLTP ( MySQL, PostgreSQL, SQL Server, MariaDB, Aurora, Oracle)
  • DynamoDB - Non relational DB
  • Redshift - OLAP - Data warehousing system
  • ElastiCache - Cloud in-memory DB (Redis, Memcache)
  • DMS - Database migration service

OLTP vs. OLAP

  • OLTP (On-line Transaction Processing) is characterized by a large number of short on-line transactions (INSERT, UPDATE, DELETE). The main emphasis for OLTP systems is put on very fast query processing, maintaining data integrity in multi-access environments and an effectiveness measured by number of transactions per second. In OLTP database there is detailed and current data, and schema used to store transactional databases is the entity model (usually 3NF).

  • OLAP (On-line Analytical Processing) is characterized by relatively low volume of transactions. Queries are often very complex and involve aggregations. For OLAP systems a response time is an effectiveness measure. OLAP applications are widely used by Data Mining techniques. In OLAP database there is aggregated, historical data, stored in multi-dimensional schemas (usually star schema).

RDS (Relational Database Service)

Amazon’s Relational Database Service is a cloud service that allows you to easily create, operate and even gauge a relational database. Right now Amazon provides 6 services under RDS.

  • MySQL
  • PostgreSQL
  • SQL Server
  • MariaDB
  • Aurora
  • Oracle
